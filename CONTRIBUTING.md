# Contribuindo para o REST Validation Starter

Obrigado por seu interesse em contribuir! Este projeto segue padrões de código limpo e modernidade.

## Como contribuir

1. **Faça um Fork** do projeto.
2. **Crie uma Branch** para sua modificação (`git checkout -b feature/minha-melhoria`).
3. **Mantenha o Padrão**:
   - Use Java 21.
   - Siga as convenções de nomeclatura em Inglês para o código.
   - Adicione testes unitários para novas funcionalidades.
4. **Documente**: Se adicionar uma configuração, atualize o `README.md`.
5. **Envie um Pull Request** detalhando o que foi feito.

## Configuração do Ambiente

- JDK 21.
- Maven 3.9+.
- O projeto usa o Spring Boot Starter Parent para gerenciar versões de teste.

## Padrão de Commits

Preferencialmente, utilize [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` para novas funcionalidades.
- `fix:` para correção de bugs.
- `docs:` para documentação.
- `test:` para testes.

Agradecemos sua ajuda!
